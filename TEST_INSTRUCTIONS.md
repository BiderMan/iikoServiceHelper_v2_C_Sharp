# Инструкция по запуску стресс-тестов для проверки решения проблемы залипания клавиш

## Запуск через консольное приложение (рекомендуется)

### 1. Запуск с 50 повторениями:
```bash
cd Tests/StressTestConsole
dotnet run 50
```

### 2. Запуск с 100 повторениями (по умолчанию):
```bash
cd Tests/StressTestConsole
dotnet run
```
или
```bash
cd Tests/StressTestConsole
dotnet run 100
```

### 3. Запуск с произвольным количеством повторений:
```bash
cd Tests/StressTestConsole
dotnet run [число_повторений]
```

## Запуск через xUnit тесты

### 1. Запуск всех тестов:
```bash
dotnet test Tests/iikoServiceHelper.Tests/
```

### 2. Запуск конкретного теста:
```bash
dotnet test Tests/iikoServiceHelper.Tests/ --filter "FullyQualifiedName~RunModifierKeyManagerStressTest_100Times"
```

## Структура тестов

### 1. `ModifierKeyManagerTests.cs`
- Базовые юнит-тесты для класса `ModifierKeyManager`
- Проверяют корректность работы методов класса

### 2. `StressTestRunner.cs`
- Стресс-тест, запускающийся 100 раз подряд
- Проверяет стабильность работы в длительной нагрузке

### 3. `StressTestConsole` (консольное приложение)
- Интерактивное консольное приложение для запуска тестов
- Поддерживает параметр количества повторений
- Показывает прогресс выполнения
- Выводит статистику по завершении

## Что тестируется

1. **Обновление состояния модификаторов**
   - Проверка корректности отслеживания состояния Alt, Ctrl, Shift

2. **Обнаружение одиночных нажатий**
   - Проверка корректности определения одиночных нажатий модификаторов

3. **Освобождение залипших клавиш**
   - Проверка работы механизма `ForceReleaseStuckKeys()`

4. **Освобождение всех модификаторов**
   - Проверка работы механизма `ForceReleaseAllModifiers()`

5. **Синхронизация состояний**
   - Проверка корректной синхронизации между внутренним и физическим состоянием клавиш

## Ожидаемые результаты

- Все тесты должны проходить успешно
- Количество ошибок должно быть равно 0
- Не должно быть залипших модификаторов после выполнения операций
- Механизм коррекции должен корректно срабатывать при необходимости

## Интерпретация результатов

- **Зеленый цвет в консоли**: Все тесты пройдены успешно
- **Красный цвет в консоли**: Обнаружены ошибки
- **Сообщения в логе**: Информация о работе механизма коррекции

## Примечания

- Тесты имитируют интенсивное использование модификаторов
- Механизм коррекции будет срабатывать при обнаружении расхождений
- В нормальной ситуации количество срабатываний коррекции должно быть минимальным
