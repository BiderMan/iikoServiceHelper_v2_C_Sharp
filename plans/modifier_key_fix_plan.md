# План улучшения механизма предотвращения залипания клавиш

## Проблема
В приложении AutoHutKey возникает проблема с залипанием клавиш (stuck keys), особенно с модификаторами (Alt, Ctrl, Shift). В логах видны сообщения "Global Alt Hook: Lonely Alt-up detected. Passing event through to prevent stuck keys", что указывает на частичную работу механизма, но не на полное решение проблемы.

## Анализ текущей реализации

### Компоненты системы:
1. `HotkeyManager.cs` - основной обработчик горячих клавиш
2. `AltBlockerService.cs` - дополнительный сервис для предотвращения залипания Alt
3. `CommandExecutionService.cs` - содержит методы `SafeReleaseModifiers()` и `ForceReleaseAlt()`
4. `NativeMethods.cs` - нативные методы для работы с клавиатурой

### Текущие механизмы:
1. Отслеживание физического состояния клавиш через `GetAsyncKeyState`
2. Внутренние флаги состояния модификаторов
3. Принудительное освобождение клавиш при выполнении команд
4. Периодическая проверка состояния

## Причины проблем

1. **Конкурирующие хуки**: Два разных хука клавиатуры могут конфликтовать
2. **Рассинхронизация состояний**: Внутреннее состояние может не совпадать с физическим
3. **Недостаточная обработка всех модификаторов**: Основное внимание уделяется Alt, но Ctrl и Shift также могут залипать
4. **Недостаточная синхронизация между компонентами**: Разные части приложения имеют разные представления о состоянии клавиш

## Предлагаемое решение

### 1. Централизованное управление модификаторами
Создать единый класс `ModifierKeyManager`, который будет:
- Управлять состоянием всех модификаторов (Alt, Ctrl, Shift)
- Синхронизировать внутреннее и физическое состояние
- Обеспечивать согласованность между всеми компонентами

### 2. Улучшенный алгоритм детекции залипших клавиш
- Периодическая проверка физического состояния с интервалом 50мс
- Сравнение с внутренним состоянием
- Автоматическое исправление расхождений

### 3. Интеграция с существующими компонентами
- Интеграция с `HotkeyManager` для получения уведомлений о нажатиях
- Использование в `CommandExecutionService` для безопасного выполнения команд
- Замена функциональности `AltBlockerService`

### 4. Улучшенный механизм освобождения
- Более точное определение залипших клавиш
- Последовательное освобождение с учетом всех вариантов каждой клавиши (левая/правая/общая)

## Архитектурные изменения

### Новый класс: `ModifierKeyManager`
```csharp
public class ModifierKeyManager : IDisposable
{
    // Состояния модификаторов
    private volatile bool _isAltDown = false;
    private volatile bool _isCtrlDown = false;
    private volatile bool _isShiftDown = false;
    
    // Физическое состояние
    private volatile bool _physicalAltDown = false;
    private volatile bool _physicalCtrlDown = false;
    private volatile bool _physicalShiftDown = false;
    
    // Методы:
    // - UpdateModifierState() - обновление внутреннего состояния
    // - IsLoneModifierPress() - проверка одиночного нажатия
    // - ForceReleaseStuckKeys() - освобождение залипших клавиш
    // - ForceReleaseAllModifiers() - освобождение всех модификаторов
}
```

### Интеграция с `HotkeyManager`
- Добавить ссылку на `ModifierKeyManager`
- Обновлять состояние модификаторов при получении событий
- Использовать для определения одиночных нажатий

### Интеграция с `CommandExecutionService`
- Использовать для безопасного выполнения команд
- Заменить текущие методы `SafeReleaseModifiers()` и `ForceReleaseAlt()`

### Улучшение `AltBlockerService`
- Либо заменить полностью на `ModifierKeyManager`
- Либо адаптировать для использования нового менеджера

## Потенциальные риски

1. **Конфликты с другими приложениями**: Агрессивное освобождение клавиш может конфликтовать с другими приложениями
2. **Производительность**: Постоянная проверка состояния каждые 50мс может влиять на производительность
3. **Несовместимость**: Изменения могут повлиять на существующую логику обработки клавиш
